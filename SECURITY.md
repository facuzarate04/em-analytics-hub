# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in em-analytics-hub, please report it responsibly.

**Do not open a public issue for security vulnerabilities.**

Instead, please email: **hola@r4mus.com**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response

You can expect an initial response within 48 hours. We will work with you to understand the issue and coordinate a fix before any public disclosure.

## Scope

This policy covers the em-analytics-hub plugin code. For issues with EmDash core, please report to the EmDash project directly.

## Privacy

This plugin is designed with privacy as a core principle:

- No cookies or localStorage
- No fingerprinting
- Daily-rotating IP hashes
- DNT (Do Not Track) respected
- Bot filtering
- Configurable excluded paths and IPs

If you find any behavior that contradicts these privacy principles, please report it.
